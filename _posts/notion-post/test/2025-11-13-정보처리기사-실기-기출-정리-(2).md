7. EAI(Enterprise Application Integration)의 구축 유형에 대한 설명이다. 괄호(①, ②) 안에 각각 들어갈 알맞은 유형을 쓰시오.

	EAI(Enterprise Application Integration)는 기업 내 각종 애플리케이션 및 플랫폼 간의 정보 전달, 연계 통합 등 상호 연동이 가능하게 해주는 솔루션이다. 비즈니스 간 통합 및 연계성을 증대시켜 효율성 및 각 시스템 간의 확정성(Determinacy)을 높여 준다. EAI의 구축 유형은 다음과 같다.


	<div class="callout">

	- (<span class="blank">①</span>)
		- 가장 기본적인 애플리케이션 통합 방식으로, 애플리케이션을 1 : 1로 연결한다.
		- 변경 및 재사용이 어렵다.

		<img src="/assets/img/notion-post/test/정보처리기사-실기-기출-정리-(2)/0.png" alt="image-0" class="normal">

	- (<span class="blank">②</span>)
		- 단일 접점인 허브 시스템을 통해 데이터를 전송하는 중앙 집중형 방식이다.
		- 확장 및 유지 보수가 용이하다.
		- 허브 장애 발생 시 시스템 전체에 영향을 미친다.

		<img src="/assets/img/notion-post/test/정보처리기사-실기-기출-정리-(2)/1.png" alt="image-1" class="normal">

	- Message Bus
		- 애플리케이션 사이에 미들웨어를 두어 처리하는 방식이다.
		- 확장성이 뛰어나며 대용량 처리가 가능하다.

		<img src="/assets/img/notion-post/test/정보처리기사-실기-기출-정리-(2)/2.png" alt="image-2" class="normal">

	- Hybrid
		- 그룹 내에서는 (<span class="blank">②</span>) 방식을, 그룹 간에는 Message Bus 방식을 사용한다.
		- 필요한 경우 한 가지 방식으로 EAI 구현이 가능하다.
		- 데이터 병목 현상을 최소화할 수 있다.

		<img src="/assets/img/notion-post/test/정보처리기사-실기-기출-정리-(2)/3.png" alt="image-3" class="normal">


	</div>

8. 소프트웨어 인터페이스 구현에 관련된 다음 설명에서 괄호에 공통으로 들어갈 알맞은 용어를 쓰시오.

	(<span class="blank"></span>)은 속성-값 쌍(Attribute-Value Pairs)으로 이루어진 데이터 객체를 전달하기 위해 사람이 읽을 수 있는 텍스트를 사용하는 개방형 표준 포맷이다. 비동기 처리에 사용되는 AJAX에서 XML을 대체하여 사용되고 있다. 다음은 (<span class="blank"></span>)이 가질 수 있는 자료 기본형이다.


	| 속성   | 특징                                       |
	| ---- | ---------------------------------------- |
	| 숫자   | 정수와 실수를 표현한다.                            |
	| 문자열  | 유니코드 문자로 표현하며, 큰따옴표(`" "`)로 묶는다.         |
	| 참/거짓 | 참(True) 또는 거짓(False)을 표현한다.              |
	| 배열   | 다양한 요소들을 쉼표로 구분하여 표현하며, 대괄호(`[ ]`)로 묶는다. |
	| 객체   | 이름/값 쌍으로 표현하며, 증괄호(`{ }`)로 묶는다.          |
	| NULL | 아직 알려지지 않거나 모드는 값을 표현하기 위한 자료형이다.        |


	<details>


	<summary markdown="span"><ins>답 확인하기</ins></summary>


	JSON (JavaScript Object Notation)


	</details>
